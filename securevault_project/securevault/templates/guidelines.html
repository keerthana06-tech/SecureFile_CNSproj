User Guidelines

Everything you need to know about using SecureVault safely and effectively.

Voice Assistant

Click Read Guidelines to have the guide read aloud.

1. Creating an Account

Visit the Sign Up page and fill in your username, email address, and a strong password. Your password must be at least 6 characters. Once registered, you can log in using your credentials. Use the Remember Me option to stay logged in across browser sessions.

2. File Tamper Detection

Go to the Tamper Detection page from the sidebar. In Step 1, upload your original file. The system will generate a SHA-256 hash — a unique digital fingerprint. Then in Step 2, upload the file you wish to compare. The system will compare hashes instantly. If hashes match, the file is unmodified. If they differ, tampering is detected. For images, highlighted bounding boxes show the exact changed regions. For text files, a line-by-line diff is shown.

3. Using the Secure Vault

The Secure Vault encrypts all your files using AES-256 Fernet encryption before storing them. No one — not even the server administrators — can read your files without your credentials. To upload, click Upload File, select a file and destination folder, then click Encrypt and Upload. To access a file, click the Access button. You will be prompted for your account password first. After verification, a one-time password (OTP) is generated. Enter the OTP to decrypt and download your file.

4. Folder Management

You can organize your vault files into custom folders. Click New Folder and enter a folder name. When uploading, select the target folder from the dropdown. You can delete folders at any time — note that deleting a folder does not delete the files within it; they will remain accessible.

5. Security Best Practices

Use a strong, unique password that includes upper and lower case letters, numbers, and symbols. Never share your OTP with anyone. Always log out when using shared computers. Regularly scan your important files for tampering. Store sensitive documents exclusively in the encrypted vault. Enable Remember Me only on personal, trusted devices.

6. Understanding Hash Values

SHA-256 produces a fixed 64-character hexadecimal string for any file. Even a single changed byte produces a completely different hash. This makes it impossible to modify a file without detection. You can view the hash of any scanned file in the Scan History table on the Tamper Detection page.
